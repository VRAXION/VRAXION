# Support

This repository is the public SDK and documentation review surface for
VRAXION. Support here is limited to public files and public releases, public
workflow behavior, and public documentation.

## Use Public Issues For

- broken public links or missing public docs
- public crate metadata, examples, or build instructions
- GitHub Pages behavior visible from the public site
- public release notes, artifact names, checksums, or claim wording
- public guard script or CI behavior

## Do Not Include

Do not put secrets, tokens, private code, internal implementation excerpts,
non-public training data, raw operator output, local machine paths, production
config, private dashboards, or credential values in public issues, pull
requests, comments, or discussions.

If a report needs private material to explain the impact, do not open a public
issue. Use the security route or contact the repository owner profile instead.

## Security Reports

Use `SECURITY.md` for vulnerabilities, suspected secret exposure, or reports
with security impact. Security-sensitive reports should describe credential
types and impact without pasting live credential values.

## Release Or Artifact Questions

Use `PUBLIC_RELEASE_CHECKLIST.md` before proposing a new public artifact,
download path, release status change, checksum update, or public claim change.
Use `PUBLIC_GITHUB_STATE.md` before relying on a GitHub release, tag, asset, or
Pages deployment as current public status.
Maintainers can answer support questions only from public files and public
releases unless a separate private review path has been opened.
