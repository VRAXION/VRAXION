SYSTEM // INSTNCT

INSTNCT is the current product name for the architecture previously codenamed PRIME C‑19. It is a pointer‑driven recurrent memory system operating on a continuous circular manifold (ring state), designed to remain seam‑safe under closed‑loop transport. Note: earlier Zenodo snapshots/DOIs may still use the PRIME C‑19 name.

1. INSTNCT (RESEARCH PREVIEW)

Current Snapshot: Zenodo v0.5.3 (Jan 2026)

  • Core: Pointer‑driven read → update → write on a circular ring state.
  • Runtime: Python + PyTorch (prototype).
  • Objective: Stable long‑horizon learning without seam tearing.

2. SEAM-AWARE TRANSPORT

Topology: Circular winding logic prevents "tearing".

  • Explicit handling of circular wrap.
  • Shortest-arc interpolation for state continuity.

3. POINTER DYNAMICS

Stabilization: Reducing numerical noise in early regimens.

  • Walk/Jump dynamics with inertial stabilizers.
  • Prevents pathological stalls during traversal.

4. INFERENCE EFFICIENCY (HYPOTHESIS TARGET)

Hypothesis: if implemented and optimized correctly, pointer‑routed recurrence on a structured ring may enable large inference efficiency gains (potentially orders of magnitude) by leveraging locality and reuse. Not proven. Not a benchmark claim.

  • Status: current code is a research prototype; training still demands compute.
  • What would count as evidence: reproducible profiling at matched quality vs baselines.
  • Plan: publish speed/accuracy curves + negative controls (no “magic numbers”).

1. REPRODUCIBILITY

Trust but Verify: Benchmarks are meaningless without reproduction.

  • Run the minimal scripts (see README).
  • Confirm baseline behavior before trusting metric shifts.

2. NEGATIVE CONTROLS

Ablation Testing: We document what happens when components are removed.

  • "If it still works without X, X wasn't the point."
  • Documenting failure modes when stabilizers are disabled.

3. ADVERSARIAL CHECKLIST

Stress Testing: The fastest path to truth is trying to break it.

  • Does behavior drift across long runs?
  • What happens at high jump rates?
  • Is there hidden leakage ("cheats")?

4. INTEGRITY STANDARD

The Rigor Rule:

  • MEASURED: Reproducible by an outsider.
  • OBSERVED: Internal result only. Treat with caution.
  • "If it can't be reproduced, it's not a measurement."

HYPOTHESIS // UNPROVEN

This section documents ideas that do not exist as established science. It is here to timestamp concepts and propose falsifiable tests.

0. PILOT‑PULSE CONJECTURE

Hypothesis: intelligence is not only compute or storage — it’s navigation performance (Pilot) over a learned structure (Substrate). Speculative framing; not a consciousness claim.

  • Strong substrate, weak pilot: memory exists, routing collapses.
  • Strong pilot, weak substrate: stable traversal, low capacity.
  • Aligned pilot + substrate: stable long-horizon learning becomes possible.

1. THEORY OF THOUGHT

Hypothesis: "Thought" is controlled traversal over a structured latent manifold.

  • Identity: Continuity under transport (no seam tearing).
  • Insight: Finding a low-energy path to a new stable basin.

2. SELF-MODELING LOOP

Hypothesis: If a system predicts its own near-future internal state, "self-monitoring" signals emerge.

  • Test: Does self-forecasting accuracy rise over training?
  • Prediction: Self-prediction degrades if continuity is broken.

3. TERMINAL LUCIDITY (ANALOGY)

Concept: "Lucidity bursts" appear when constraints allow re-entry into a stable trajectory after degradation.

  • Experiment: Degrade system -> Open stability window -> Measure return.
  • Note: Not a medical claim. Purely AI architecture analogy.

4. TOPOLOGICAL RECURSION

Hypothesis: repeatedly applying a looped operator (read → update → write → move) can form stable recurrence structures (dwell, cycles, attractors) that behave like “thought trajectories”.

  • Observable signatures: dwell time, flip rate, ABAB ping‑pong, mode entropy.
  • Falsifiable: if no stable recurrence appears under controlled conditions, the framing fails.
