# Operator Library Cards

This public file is a compact taxonomy of the Operator library. The full
frontier card ledger, exact activation traces, mutation history, and private
training evidence are not part of the public release surface.

## Boundary

```text
canonical generic term = Operator
legacy alias = Pocket
public status = scoped evidence summaries only
```

These cards do not claim open-domain reasoning, production assistant behavior,
PermaCore memory, or unrestricted language understanding.

## Public Operator Families

| Family | Public Role | Example Scope |
|---|---|---|
| Lens | Detects bounded structural cues. | Evidence span, contrast, missing dependency. |
| Guard | Blocks unsafe or unsupported commits. | Unsupported answer, stale trace, wrong scope. |
| Resolver | Chooses among bounded competing proposals. | Source priority, latest-span resolution. |
| Scribe | Produces bounded structured output. | Citation link, grounded answer template. |
| Normalizer | Maps visible surface forms to canonical internal forms. | Unicode operator or alias normalization. |
| Router | Selects a scoped route under Agency control. | Visible calculation trace route. |

## Representative Public Cards

### CALC-SCRIBE

```text
scope = visible calculation-trace validation
claim = validates visible expression/result markers under scoped checks
non-claim = does not solve hidden word problems or arbitrary math text
```

### Evidence Conflict Detector

```text
scope = detects bounded conflict cues in visible evidence
claim = can propose defer/resolve routes when conflict structure is visible
non-claim = does not prove which source is globally true
```

### Unsupported Answer Guard

```text
scope = blocks answer commits when evidence coverage is insufficient
claim = reduces unsupported answer commits inside governed routes
non-claim = does not guarantee factual truth outside the checked route
```

### Output Trace Scribe

```text
scope = renders bounded trace/citation structure
claim = can preserve checked evidence links in scoped outputs
non-claim = does not create factual support that is absent from input
```

## Public Use

Use this file to understand the public role taxonomy. Do not use it as a full
Operator registry, promotion ledger, or training reconstruction guide.
