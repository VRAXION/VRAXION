Research

This surface is the public research snapshot for Vraxion: what the project is trying to prove now, how claims are judged, and which recent turns still matter.

It is not the full archive. The deep chronology, retained reversals, and protocol residue stay on the wiki's Research Process & Archive.

Current frame

The stable public release is still v4.2.0. The biggest unresolved pressure is no longer basic trainability, but whether language evaluation, seed variance, and context-dependent task learning survive repeated adversarial reruns.

How claims are judged

Public discipline matters as much as raw result headlines.

Evidence minimum

run_cmd.txt, env.json, metrics.json, and summary.md remain the minimum bundle behind meaningful claims.

Current open gates

These are the gates that still separate promising lines from stronger public promotion.

Gate Why it is still open Current pressure
Language robustness Single strong runs are not enough if the band collapses under reruns or multi-window evaluation. Repeated adversarial reruns, cleaner eval contracts, and clearer public metrics.
Seed stability The landscape is visibly rugged, which means single-seed stories overstate certainty. Seed audits, reranking, and multi-window discipline.
Context-dependent task learning The architecture claim is stronger if task memory survives outside toy or fragile settings. Windowed input injection, associative memory, and stronger nontrivial task eval.
Rust search ceiling The tested 1+1 ES setup still converges into a stable band instead of a durable breakthrough. Depth lines, different exploration regimes, and recipe parity.

Milestone rail
