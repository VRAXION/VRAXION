# STABLE_LOOP_PHASE_LOCK_048_EXTERNAL_HELDOUT_EVAL_SUITE Contract

## Summary

047 showed that concrete train -> inference behavior stays stable across
checkpoint time in the bounded runner suite. 048 tests whether that behavior
survives a frozen heldout/OOD eval corpus that is committed as data and loaded
by the runner, rather than generated by the same active eval call.

This is not a new capability probe and not a scaleout launch. It does not
enable production default training, promote public beta, or claim full VRAXION,
language grounding, or consciousness.

## Core Question

```text
Does the 046/047 input-conditioned inference behavior remain positive on a
frozen heldout/OOD eval suite with no train leakage, template holdout, family
holdout, and shortcut controls?
```
