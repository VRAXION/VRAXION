Rust Implementation Surface

Rust

The Rust lane is now a serious parallel implementation surface for INSTNCT, not just a fast forward-pass port.

Core systems work is largely in place. The live frontier is language-eval robustness, seed stability, and breaking the stable 17-18% convergence band of the tested 1+1 ES setup.

What is already solid

Owned Network, rollback snapshots, mutation API, CSR acceleration, checkpoint persistence, and a learnable int8 readout are all already real in the public Rust line.

Implementation status

The library is already beyond "forward-pass demo" status.

Core substrate

Owned topology, reusable state, CSR cache, propagation, and mutation-selection now live inside one coherent library surface.

Persistence

Genome save/load and checkpoint bundles allow resumable, adversarially testable longer runs.

Readout and input

Learnable int8 projection and SDR table are already integrated into the live implementation lane.

What still remains

Full English recipe parity, stronger exploration than plain 1+1 ES, and better constrained depth lines such as pocket chains.

Performance and frontier

The current problem is no longer "can Rust be fast?" but "what actually breaks the search ceiling?"

Current decisions

  • u32/i32 hot path, compact indices, and SoA layout remain the best tested runtime choices.
  • CSR skip-inactive and deterministic benchmark policy are accepted defaults.
  • Bitpacking, AVX2 shortcuts, and other micro-optimization narratives are not promoted unless they survive deterministic reruns.

Current frontier

Stable 17-18% behavior looks like a structural exploration limit of the tested setup, not a missing one-line hyperparameter fix.

Depth lines such as pocket chains, better exploration than plain 1+1 ES, and recipe-level English parity remain the active paths that could change the public Rust story.
